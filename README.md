# 5G通信基建数智化交付系统 (Demo 版)

这是一个基于 Streamlit 的单页面 Demo 应用，用于演示“基站设计元数据表 -> 规则引擎校验 -> 大模型生成 -> 施工 BOM 清单与工序指导书”的自动化转化流程。

当前版本默认使用本地 `mock_llm_response(data, mode)` 模拟 AI 生成结果，便于无密钥演示；同时预留了 OpenAI 兼容的 Chat Completions 接口配置，可在侧边栏切换为真实大模型 API。

## 功能概览

- 上传 `.xlsx` 或 `.csv` 格式的基站设计元数据表
- 支持三种生成模式：生成 BOM 清单、生成工艺指导书、全量生成
- 未上传文件时自动展示 5 行模拟基站数据
- 使用 Pandas 读取和预览前 5 行原始数据
- 对上传数据做必填字段和线缆距离校验
- 使用规则引擎估算电源线、光缆、接地线、标签、桥架和防水套件数量
- 支持本地 Mock 或真实大模型 API 两种生成引擎
- 对大型工程表自动生成工程摘要，只向模型发送摘要和样本行，降低接口超时风险
- 真实 API 模式支持超时时间、样本行数和失败回退配置
- 生成过程提供数字化进度反馈，展示数据校验、规则估算、AI 生成和报告封装阶段
- 生成通信施工文档风格的结果，包含物料统计、安全注意事项和验收要点
- 提供 Demo 版下载按钮，生成 Word 可打开的 `.doc` 报告文件和 Excel `.xlsx` 交付清单

## 界面与交互

当前界面保持 Streamlit 单页面 Demo 形态，主要交互集中在左侧浅色控制台，主区域按交付流程组织：

- 顶部以中文化品牌 Hero 展示“云筑交付中枢”，并提供“查看工程工作台”和“生成交付报告”两个真实操作入口
- 左侧控制台按数据接入、生成设置、真实 API 配置和执行分组收纳
- 主区域使用高端 dashboard 风格的视觉 Hero、状态条和 bento 工作流模块
- 数据预览与数据质量校验采用双栏布局，便于演示时快速确认输入质量
- 规则指标总览展示记录数、线缆总长、字段数和生成模式
- 生成结果区域提供进度反馈、报告预览和 Word 下载
- 报告预览按 Taste Skill 的高端 dashboard 方向拆分为工程概况、BOM、质量验收和完整报告页签，并在未生成正式报告前展示数据化演示预览
- 工程概况使用指标模块呈现站点规模、路由长度、设备型号、取电方式和风险重点
- BOM 使用结构化表格呈现物料名称、规格、估算数量、单位和施工用途
- 质量验收使用检查清单模块呈现设备安装、线缆工艺、电源接入、接地、光链路和资料闭环

## 数据字段

Demo 内置数据和推荐上传表头如下：

| 字段 | 说明 |
|---|---|
| 站点编号 | 基站或站点唯一编号 |
| 站点类型 | 如宏站、楼面站、室分站、微站 |
| AAU型号 | AAU 或 pRRU 设备型号 |
| BBU型号 | BBU 设备型号 |
| 线缆敷设距离 | 线缆路由长度，单位默认为米 |
| 取电方式 | 市电直供、交流配电箱、弱电井取电等 |

## 本地运行

建议使用 Python 3.10 及以上版本。

```bash
pip install -r requirements.txt
streamlit run app.py
```

启动后在浏览器访问 Streamlit 输出的本地地址，通常是：

```text
http://localhost:8501
```

## 真实大模型 API 配置

默认情况下无需 API Key，直接使用本地 Mock 结果。

如果要接入真实大模型，在侧边栏选择：

```text
AI引擎 -> 真实大模型API（OpenAI兼容）
```

然后填写：

| 配置项 | 说明 |
|---|---|
| API地址 | 兼容 Chat Completions 的接口地址 |
| 模型名称 | 模型 ID，如平台提供的具体模型名 |
| API Key | 对应平台的访问密钥 |
| API超时时间 | 默认 180 秒，大型工程文件建议 180-300 秒 |
| 送入模型的样本行数 | 默认 30 行，系统会额外提供全量工程摘要 |

也可以通过环境变量预设：

```bash
set LLM_API_URL=https://api.openai.com/v1/chat/completions
set LLM_MODEL=gpt-4.1-mini
set LLM_API_KEY=你的API密钥
set LLM_TIMEOUT=180
set LLM_SAMPLE_ROWS=30
```

## 大型工程文件处理策略

大型 Excel/CSV 文件不建议逐行直接发送给大模型，否则容易出现接口超时、上下文过长或响应缓慢。本 Demo 的处理方式是：

1. Pandas 读取全量设计元数据。
2. 规则引擎基于全量数据计算 BOM 估算值。
3. 系统生成站点类型、AAU/BBU 型号、取电方式、线缆距离、长路由站点等工程摘要。
4. 外接 API 只接收工程摘要和少量样本行。
5. 如果 API 超时，界面会提示原因；开启“API失败时自动生成本地工程报告”后会自动回退到规则引擎报告。

这能保证大型数据场景下页面不会直接抛出 Python 堆栈，也能让用户通过数字化进度条判断任务是否仍在运行。

## Demo 说明

本项目是演示版本，重点验证交互流程和业务表达：

- 默认大语言模型调用为本地 Mock 函数，真实 API 为可选配置
- “下载 Word 交付报告”当前生成 Word 可打开的 `.doc` 文件，“下载 Excel 交付清单”生成多 Sheet `.xlsx` 工作簿，暂不生成真实 PDF
- BOM 数量采用简化经验规则估算，不作为真实工程采购依据

后续可继续扩展模板化 Word/PDF 导出、更完整的 BOM 规则库、项目级交付记录管理和权限控制。
